# Terms

Fixed system terms for PAOS:

- `system`: the full PAOS knowledge operating system
- `module`: an independently maintainable unit of knowledge
- `state`: current active truth that should load by default
- `knowledge`: durable reusable understanding
- `report`: bounded synthesized output for a specific question
- `template`: reusable structural contract for a document or machine artifact
- `manifest`: machine-readable registry and dependency descriptor
- `roadmap`: intended future structural change
- `archive`: inactive historical material outside default context
- `canonical`: the single source of truth for a concept
- `freshness`: whether a state or report is current enough for default use
