# Manifest Template Design

Required fields:
- `moduleId`
- `title`
- `purpose`
- `scope`
- `dependsOn`
- `defaultLoad`
- `status`

Optional fields:
- `inputs`
- `outputs`
- `canonicalFiles`
- `freshnessRule`
- `replacementOf`

Rules:
- one manifest per module scope
- current truth only
- no historical discussion blocks
